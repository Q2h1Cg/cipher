Bleichenbacher's e=3 RSA Attack
题目描述：
指数是3的RSA加密是十分流行的，因为计算速度很快
加密过程：对数字取立方然后对公共模数取模
c = m ** 3 % n
对于低指数的RSA，唯一需要考虑的是进行立方运算后的信息块，不能足够大来隐藏模数的信息。
当RSA用来签名时，操作方法是相反的，认证者进行立方运算来解密消息。
当使用RSA进行签名时，需要提供包括消息摘要的分组，然后根据PKCS1.5标准来进行填充例：
00h 01h ffh ffh ... ffh ffh 00h ASN.1 GOOP HASH
如上例所示。对于RSA的认证者来说他们的认证过程就是解密消息并且通过寻找padding来分析，但是存在一个缺陷即这个认证过程没有检查所有的padding，如果在中间的填充的ff中伪造看起来合法的认证消息，也能够认证成功。
所以根据这个原理来为字符串“hi mom”伪造RSA签名。
